# 高质量角色卡和世界书提取方案

## 问题背景

高质量的角色卡和世界书是小说生成的核心基础。如何从现有内容中提取、整理和生成高质量的角色卡和世界书，是 NovelForge 系统的关键问题。

## 解决方案

### 1. 角色卡提取与生成

#### 1.1 从现有小说内容中提取角色信息

**自动提取流程：**
```
小说文本 → NLP分析 → 角色识别 → 信息提取 → 结构化存储
```

```typescript
interface CharacterExtraction {
  name: string
  mentions: number          // 出现次数
  contexts: string[]       // 出现场景
  relationships: Map<string, string>  // 与其他角色的关系
  traits: string[]          // 性格特征
  dialogues: string[]       // 对话样本
  actions: string[]         // 行为描述
}

class CharacterExtractor {
  async extractFromText(text: string): Promise<CharacterExtraction[]> {
    //1. 使用 AI 识别角色
    const characters = await this.identifyCharacters(text)
    
    //2. 提取每个角色的详细信息
    const results = await Promise.all(
      characters.map(char => this.extractCharacterInfo(text, char))
    )
    
    return results
  }
  
  private async identifyCharacters(text: string): Promise<string[]> {
    const prompt = `
请分析以下文本，识别出所有角色名称：

【文本】
${text}

【要求】
1. 列出所有出现的角色名称
2. 按出现频率排序
3. 排除非角色实体（如地名、物品名）

【角色列表】
    `
    
    const response = await this.callAI(prompt)
    return this.parseCharacterNames(response)
  }
  
  private async extractCharacterInfo(
    text: string,
    characterName: string
  ): Promise<CharacterExtraction> {
    const prompt = `
请分析角色"${characterName}"在以下文本中的信息：

【文本】
${text}

【分析要求】
1. 性格特征（3-5个关键词）
2. 外貌描述
3. 背景故事
4. 与其他角色的关系
5. 典型对话（2-3句）
6. 行为模式

【分析结果】
    `
    
    const response = await this.callAI(prompt)
    return this.parseCharacterInfo(response, characterName)
  }
}
```

```typescript
class HybridCharacterGenerator {
  async generate(
    sourceText: string,
    userRequirements: UserRequirements
  ): Promise<HighQualityCharacterCard> {
    //1. 从源文本提取基础信息
    const extracted = await this.extractFromText(sourceText)
    
    //2. 结合用户需求生成完整角色卡
    const prompt = this.buildHybridPrompt(extracted, userRequirements)
    const generated = await this.callAI(prompt)
    
    //3. 质量评估
    const quality = await this.evaluateQuality(generated)
    
    //4. 如果质量不达标，进行优化
    if (quality.overall < 70) {
      return await this.optimize(generated, quality.suggestions)
    }
    
    return generated
  }
  
  private buildHybridPrompt(
    extracted: CharacterExtraction,
    requirements: UserRequirements
  ): string {
    return `
【角色生成任务 - 混合模式】

出现次数：${extracted.mentions}
```

#### 4.2 知识图谱构建

```typescript
class KnowledgeGraphBuilder {
  buildCharacterGraph(characters: HighQualityCharacterCard[]): Graph {
    const graph = new Graph()
    
    characters.forEach(char => {
      graph.addNode(char.name, {
        type: 'character',
        data: char
      })
      
      char.relationships.forEach(rel => {
        graph.addEdge(char.name, rel.character, {
          type: rel.type,
          description: rel.description
        })
      })
    })
    
    return graph
  }
  
  buildWorldGraph(world: HighQualityWorldBook): Graph {
    const graph = new Graph()
    
    // 添加地点节点
    world.locations.forEach(loc => {
      graph.addNode(loc.name, {
        type: 'location',
        data: loc
      })
      
      loc.connections.forEach(conn => {
        graph.addEdge(loc.name, conn, {
          type: 'connection'
        })
      })
    })
    
    return graph
  }
}
```

#### 5.1 批量导入工具

```typescript
class BatchImporter {
  async importFromFiles(files: File[]): Promise<ImportResult> {
    const results = {
      characters: [] as HighQualityCharacterCard[],
      worlds: [] as HighQualityWorldBook[],
      errors: [] as string[]
    }
    
    for (const file of files) {
      try {
        const content = await this.readFile(file)
        const type = this.detectFileType(content)
        
        if (type === 'character') {
          const character = await this.processCharacter(content)
          results.characters.push(character)
        } else if (type === 'world') {
          const world = await this.processWorld(content)
          results.worlds.push(world)
        }
      } catch (error) {
        results.errors.push(`${file.name}: ${error.message}`)
      }
    }
    
    return results
  }
}
```
