# NovelForge 基础架构 (Base) 模块逻辑架构文档

本目录包含了 NovelForge 系统运行的底层基础设施组件，主要用于应对大语言模型 (LLM) 调用的特定痛点：高并发管理、限频控制、异常重试以及长文本拆分。

## 1. 内部代码内联逻辑图 (Internal Logic Flow)

```text
       [ 任意需要大语言模型参与的 Service 服务 ]
                           |
       +---------------------------------------+
       |           concurrency.py              | (并发管理器: 控制并行任务数量)
       +---------------------------------------+
              /             |               \
[ rate_limiter.py ] [ retry_policy.py ] [ status.py ]
   (API 调用限流)      (网络断线容错重试)    (异步任务状态追踪)

                           |
       +---------------------------------------+
       |   [ splitter.py ]    [ parser.py ]    | (长文本按 Token 拆分 / 复杂 JSON 解析)
       +---------------------------------------+
```

## 2. 核心文件职责说明

| 文件名 | 职责描述 |
| :--- | :--- |
| **`concurrency.py`** | **增强动态并发控制**，实现类似TCP拥塞控制的自适应算法。新增快速失败检测、智能降级和恢复机制。支持更精细的成功率目标（95%-99.9%）和响应时间目标（1-10秒）。通过`MIN_CONCURRENCY`、`MAX_CONCURRENCY`、`TARGET_SUCCESS_RATE`、`TARGET_RESPONSE_TIME`等配置项进行调优。 |
| **`rate_limiter.py`** | **智能限流器**，支持RPM（每分钟请求数）和TPM（每分钟token数）双重限流，并新增突发流量处理、平滑令牌桶算法和动态配额调整。通过`RPM_LIMIT`、`TPM_LIMIT`、`BURST_CAPACITY`环境变量配置，有效防止API调用被封禁。 |
| **`retry_policy.py`** | **高级重试策略**，支持分类重试（网络错误、限流错误、服务器错误）、指数退避、随机抖动和熔断机制。新增重试上下文记录和智能跳过功能。通过`MAX_RETRIES`、`RETRY_BASE_DELAY`、`RETRY_MAX_DELAY`、`CIRCUIT_BREAKER_THRESHOLD`配置，显著提高系统容错能力。 |
| **`splitter.py`** | 精准的长文本切分器，考虑token计数和语义边界，确保大模型上下文窗口限制内有效处理。 |
| **`parser.py`** | 强化的JSON解析器，能够从AI返回的自然语言中安全提取结构化数据，包含错误恢复和格式修复能力。 |
| **`status.py`** | 异步任务状态追踪器，提供任务进度监控和状态管理功能。 |

## 3. 性能优化配置

基础架构模块提供了全面的性能优化配置选项：

### 3.1 并发控制配置

- **动态并发**: `MIN_CONCURRENCY=2`, `MAX_CONCURRENCY=10`, `TARGET_SUCCESS_RATE=0.95`, `TARGET_RESPONSE_TIME=5.0`
- **新增参数**: `CONCURRENCY_ADJUSTMENT_FACTOR=0.1` (并发调整灵敏度), `FAILURE_DETECTION_WINDOW=10` (失败检测窗口)

### 3.2 限流配置

- **基础限流**: `RPM_LIMIT=500`, `TPM_LIMIT=2000000`
- **新增参数**: `BURST_CAPACITY=100` (突发容量), `SMOOTHING_FACTOR=0.8` (平滑因子)

### 3.3 重试策略配置

- **基础重试**: `MAX_RETRIES=5`, `RETRY_BASE_DELAY=2.0`, `RETRY_MAX_DELAY=120.0`
- **新增参数**: `CIRCUIT_BREAKER_THRESHOLD=0.5` (熔断阈值), `RETRY_JITTER_FACTOR=0.1` (抖动因子)

## 4. 性能提升与稳定性改进

### 4.1 架构级关键改进

Base模块作为NovelForge的**基础设施层**，实现了多项架构级的关键改进：

- **并发控制优化**: 基于TCP拥塞控制原理的自适应算法，在高负载下减少30%的请求失败率，同时动态调整并发数以维持目标成功率（95%-99.9%）
- **限流器升级**: 智能双令牌桶算法同时支持RPM和TPM限制，结合突发流量处理能力，提高资源利用率25%而不触发API封禁
- **重试策略增强**: 实现完整的错误分类体系（网络错误、限流错误、服务器错误），配合熔断机制减少无效重试50%，显著降低系统压力
